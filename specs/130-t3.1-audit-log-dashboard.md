## Cilj
Admin može da pregleda audit log kroz UI sa filterima (po korisniku, akciji, datum range-u, subject tipu). Read-only — audit log je immutable.

## Izlazi (deliverables)

### Backend
- [ ] `AdminAuditLogController::index` — listing sa filterima i paginacijom
- [ ] `AdminAuditLogController::show` — detalji jednog zapisa (full payload)
- [ ] Filter validation i query optimization (indexes na `user_id`, `action`, `subject_type`, `created_at`)
- [ ] `AuditLogPolicy` (spec 13.4: admin only read; **niko** ne može delete ili update)

### Frontend
- [ ] `resources/js/pages/admin/audit-log/index.tsx` — tabela sa filterima
- [ ] `resources/js/pages/admin/audit-log/show.tsx` — detalji
- [ ] `<AuditLogFilters>` komponenta — user picker, action enum dropdown, datum range, subject type
- [ ] `<AuditPayloadViewer>` — JSON pretty print sa syntax highlight (može lightweight, shadcn nema nativnu)

### Testovi (Pest)
- [ ] Feature: admin vidi audit log
- [ ] Feature: profesor/učenik dobija 403
- [ ] Feature: filteri rade (user, action, datum range)
- [ ] Feature: paginacija prolazi
- [ ] Feature: pokušaj DELETE/UPDATE preko endpointa vraća 405 ili 403 (immutable enforcement)
- [ ] Unit: query sa filterima koristi indexe (može EXPLAIN check u test-u za SQLite)
- [ ] Browser smoke: admin/audit-log page

## Shared edit zones
- `routes/audit.php` — **isključivo** ovaj track
- `routes/admin.php` — admin audit rute (koordinacija)
- `lang/me/audit.php` — **isključivo** ovaj track

## Acceptance criteria
- Sve Pest testove pass
- Pint clean, Wayfinder regen
- Mobile UI verifikovan
- Tabela performant za 1000+ zapisa (paginacija obavezna)
- Audit log za audit.viewed (admin pristupio dashboard) — sami sebe loguje meta-akcijom

## NE radi
- Nemoj export audit log u CSV/JSON (može u T3.2 ako vrijeme)
- Nemoj real-time stream (no Pusher / Broadcasting)
- Nemoj timeline / Gantt visualization — basic tabela je dovoljna
- Nemoj retention policy enforcement (out of scope)
- Nemoj audit log entry editing (immutable)
