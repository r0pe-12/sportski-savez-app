## Cilj
Admin može pokrenuti verifikaciju učenika kroz mock eDnevnik. Background job dohvata podatke, poredi, mijenja `Student.verification_status`. Pri nepodudaranju, prikazuje listu razlika; admin može manuelno potvrditi ili reset-ovati.

## Izlazi (deliverables)

### Backend
- [ ] `AdminStudentVerificationController` — `verify` (POST), `manualApprove` (POST), `reset` (POST), `show` (vidi detalje verifikacije)
- [ ] `VerifyStudentWithEDnevnikJob` (queue: `ednevnik`) — poziva `EDnevnikAdapter`, mapira na `EDnevnikStudentDTO`, poredi
- [ ] `EDnevnikVerificationService` — `compare(Student $local, EDnevnikStudentDTO $remote): VerificationResult`
- [ ] `VerificationResult` value object — `verified: bool`, `mismatches: array<string, [local, remote]>`
- [ ] State transitions na Student model: `unverified → pending → verified | mismatched | failed`

### Frontend (Admin only)
- [ ] `resources/js/pages/admin/students/verify.tsx` — admin pokreće verifikaciju, vidi status
- [ ] `resources/js/components/students/VerificationStatusBadge.tsx`
- [ ] `resources/js/components/students/MismatchTable.tsx` — pokazuje razlike (lokalni vs eDnevnik)

### Notifikacije (po spec 9.5)
- [ ] `StudentVerifiedNotification` — primalac: admin (in-app)
- [ ] `StudentMismatchedNotification` — primalac: admin (email + in-app, sa listom mismatches)

### Testovi (Pest)
- [ ] Feature: admin pokreće verifikaciju, FakeEDnevnik vraća verified → status verified
- [ ] Feature: admin pokreće verifikaciju, FakeEDnevnik vraća mismatched (npr. drugi razred) → status mismatched, mismatches u response-u
- [ ] Feature: admin pokreće verifikaciju, eDnevnik 404 (učenik ne postoji) → status failed
- [ ] Feature: admin manualno potvrdi mismatched → status verified, audit log entry sa "manual override"
- [ ] Feature: profesor ne može pristupiti verification rutama (403)
- [ ] Unit: `EDnevnikVerificationService::compare` detektuje mismatch u ime, prezime, razred, šifra škole
- [ ] Browser smoke: admin verifikuje učenika kroz UI

## Shared edit zones
- `routes/admin.php` — admin verifikacija rute (koordinacija sa T1.1 koji već ima base admin rute)
- `lang/me/students.php` ili `lang/me/verification.php` (zaseban fajl preporučen)
- `app/Adapters/EDnevnik/` — koristi `FakeEDnevnikAdapter` iz T1.3

## Acceptance criteria
- Sve Pest testove pass
- Pint clean, Wayfinder regen
- Audit log za student.verification_requested, student.verified, student.mismatched, student.manually_approved
- Notifikacije rade
- Mobile UI (admin može i sa telefona)

## NE radi
- Nemoj integration sa pravim eDnevnik HTTP API (FakeEDnevnik je za sad)
- Nemoj batch verifikaciju (admin verifikuje jedan po jedan)
- Nemoj public student verification status display (samo admin vidi mismatches)
- Nemoj scheduled re-verification (admin pokreće manualno)
