## Cilj
Admin može da unese rezultate za completed takmičenje. Sistem razlikuje timski sport (Result vezan za Team) i individualni (Result vezan za TeamMember). Posle unosa: Team.status `active → completed`, notifikacija profesoru i članovima.

## Izlazi (deliverables)

### Backend
- [ ] `AdminResultController` — `index` (po takmičenju), `create`, `store`, `edit`, `update`
- [ ] `StoreResultRequest`, `UpdateResultRequest` — validacija (placement int >= 1, medal_type enum, subject postoji i pripada tom takmičenju)
- [ ] `ResultEntryService::recordTeamResult(Team $team, int $placement, MedalType $medal)` — polimorfan zapis
- [ ] `ResultEntryService::recordIndividualResult(TeamMember $member, int $placement, MedalType $medal)`
- [ ] Auto-transition Team `active → completed` kad svi članovi imaju rezultat (za individualni sport) ili kad tim ima rezultat (za timski)
- [ ] `ResultPolicy` (spec 13.4: admin only)

### Frontend (Admin)
- [ ] `resources/js/pages/admin/results/index.tsx` — birati takmičenje, vidi listu ekipa/članova
- [ ] `resources/js/pages/admin/results/enter.tsx` — forma za unos (drugačije UI za team vs individual sport)
- [ ] `<ResultEntryRow>` komponenta sa placement input + medal dropdown
- [ ] Bulk save: admin može unijeti više rezultata odjednom

### Notifikacije
- [ ] `ResultEnteredNotification` — primalac: profesor + članovi (samo in-app, ne email — spec 9.5)

### Testovi (Pest)
- [ ] Feature: admin unosi timski rezultat (1 mjesto, zlato) za team, Team postaje completed
- [ ] Feature: admin unosi individualne rezultate za sve članove individual sport tima → svi članovi imaju Result, Team completed
- [ ] Feature: profesor ne može unijeti rezultate (403)
- [ ] Feature: admin može da edituje već unesen rezultat
- [ ] Feature: validation odbija placement = 0 ili negative
- [ ] Unit: polymorphic morph relacije rade — `Result::for($team)`, `Result::for($member)`
- [ ] Browser smoke: admin unosi rezultate

## Shared edit zones
- `routes/results.php` — **isključivo** ovaj track
- `routes/admin.php` — admin results rute (koordinacija)
- `lang/me/results.php` — **isključivo** ovaj track

## Acceptance criteria
- Sve Pest testove pass
- Pint clean, Wayfinder regen
- Audit log za result.entered, team.completed
- Notifikacija pojavljuje se kod profesora i u članova
- UI dovoljno usable na mobile (admin može i sa telefona)

## NE radi
- Nemoj public results page (može u T2.4 ili kasnije)
- Nemoj photo/video uploadi za rezultate
- Nemoj certificate generation za medalje (PDF) — out of scope
- Nemoj statistike (top 10 učenika, top škole) — kasnije ako bude vrijeme
- Nemoj rezultate van škole (klubovi) — spec 2 van obima
