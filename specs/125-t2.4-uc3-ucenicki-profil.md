## Cilj
Učenik može da vidi svoj profil sa fotografijom, ličnim podacima, istorijom takmičenja, osvojenih medalja. Profesor može da vidi profile učenika iz svoje škole. Admin vidi sve.

## Izlazi (deliverables)

### Backend
- [ ] `StudentProfileController` — `show`, `edit`, `update` (limited fields — student može mijenjati samo neke)
- [ ] `StudentPhotoController` — `store`, `destroy` (upload foto kroz `PrivateFileStorage`)
- [ ] `UpdateStudentProfileRequest` — različita validacija po role-u (student own — limited fields; admin — sva polja)
- [ ] `StudentHistoryService` — agregira `TeamMember` zapise sa Result-ima, sortira po datumu
- [ ] `StudentPolicy` (spec 13.4: own, school, admin)

### Frontend
- [ ] `resources/js/pages/students/profile.tsx` (own) — `/profile` shortcut za current student
- [ ] `resources/js/pages/students/show.tsx` (other view) — `/students/{student}` za profesora/admina
- [ ] `<StudentHero>` komponenta — foto, ime, škola, razred, verification badge
- [ ] `<CompetitionHistoryList>` — kartice po takmičenju sa rezultatima
- [ ] `<MedalShelf>` — vizuelni prikaz osvojenih medalja
- [ ] `<PhotoUpload>` — sa crop modal (basic, ne aggressive)

## Acceptance criteria
- Sve Pest testove pass
- Pint clean, Wayfinder regen
- Audit log za student.profile_viewed, student.profile_updated, student.photo_uploaded
- Mobile UI 360px verifikovan
- Foto prikazana kroz signed URL (TTL 5 min)
- Pristupačnost: alt text za foto, screen reader friendly history list

## NE radi
- Nemoj profile export endpoint (out of scope sa AZLP cleanup-om)
- Nemoj profile delete request workflow — admin samo soft delete iz T1.1
- Nemoj photo crop advanced features (just basic resize na server-u)
- Nemoj parental consent workflow
- Nemoj public student profile page (privacy)
