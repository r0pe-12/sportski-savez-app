## Cilj
Profesor (kroz UI iz T2.1a) može upload-ovati ljekarsku potvrdu za člana ekipe. Background job pokreće `OcrAdapter`, post-procesira odgovor, mijenja `MedicalCertificate.status`. UI prikazuje stanje u realnom vremenu (poll ili Inertia partial reload).

## Izlazi (deliverables)

### Backend
- [ ] `MedicalCertificateController` — `store` (upload), `show` (preview), `destroy` (supersede)
- [ ] `StoreMedicalCertificateRequest` — MIME validacija (PDF, JPG, PNG), max size (10 MB po spec 11.5)
- [ ] `MedicalCertificatePolicy` (spec 13.4: profesor only own team's member)
- [ ] `ValidateMedicalCertificateJob` (queue: `ocr`) — poziva `OcrAdapter`, parsira odgovor, transition `pending → valid/expired/invalid/manual_review`
- [ ] `OcrValidationService` — orkestrira post-procesiranje (regex datuma, name match)
- [ ] State machine helperi na MedicalCertificate model: `markValid`, `markExpired`, `markInvalid`, `markManualReview`, `markSuperseded`

### Frontend

## Acceptance criteria
- Sve Pest testove pass
- Pint clean, Wayfinder regen
- Browser smoke: upload prolazi, status badge se mijenja
- Audit log za certificate.uploaded, certificate.validated, certificate.invalidated, certificate.superseded
- Queue worker procesira `ValidateMedicalCertificateJob` u dev-u

## NE radi
- Nemoj integration sa pravim Google Vision API (FakeOcr je za sad)
- Nemoj manual review UI dashboard (može u T3.1 ili kasnije)
- Nemoj OCR confidence-based escalation logiku van onog što fake adapter daje
- Nemoj virus scan (S3 GuardDuty / ClamAV — out of scope)
