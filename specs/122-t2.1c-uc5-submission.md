## Cilj
Profesor (kroz UI iz T2.1a) može potpisati prijavu unosom punog imena. Sistem validira potpis, transition `draft → submitted`, generiše notifikacije profesoru i adminu. Admin može da odobri (`submitted → active`) ili odbije.

## Izlazi (deliverables)

### Backend
- [ ] `TeamController::submit` metoda (POST `/teams/{team}/submit`)
- [ ] `SubmitTeamRequest` — validacija potpisa (mora odgovarati `Auth::user()->full_name` exact match)
- [ ] `TeamRegistrationService::submit($team, $signature, $ip)` — orkestrira:
  - Provjera svi članovi imaju validne potvrde (medical_certificate.status = valid)
  - Provjera broj članova u okviru pravila sporta
  - State transition `draft → submitted`
  - Audit log entry
  - Disptach `SendTeamSubmittedNotification` job
- [ ] `AdminTeamController::approve`, `reject` (POST `/admin/teams/{team}/approve` itd.)
- [ ] `ApproveTeamRequest`, `RejectTeamRequest` (sa razlogom)

### Notifikacije (po spec 9.5)
- [ ] `TeamSubmittedNotification` — email (log driver) + database channel — primalac: profesor + admin
- [ ] `TeamApprovedNotification` — primalac: profesor
- [ ] `TeamRejectedNotification` sa razlogom — primalac: profesor

### Frontend
- [ ] `resources/js/pages/teams/review.tsx` ili `submit-step.tsx` — zadnji korak forme: pregled, potpis input field, submit dugme
- [ ] Vizuelni feedback: ako neka potvrda nije valid, dugme submit disabled + lista problema
- [ ] Posle submit: redirect na `/teams/{team}` koji prikazuje status `submitted` + očekivanje admin odluke
- [ ] Admin UI: `/admin/teams` listing sa filterom po statusu, dugme approve/reject

### Testovi (Pest)
- [ ] Feature: submit prolazi sa validnim potpisom (full name match), svi članovi valid
- [ ] Feature: submit pada ako jedan član ima invalid certifikat (UC5 alt 5a/5b enforcement)
- [ ] Feature: submit pada ako potpis ne odgovara registrovanom imenu (UC5 alt 7a)
- [ ] Feature: submit pada ako broj članova van pravila sporta
- [ ] Feature: profesor ne može submit-ovati tuđu ekipu (403)
- [ ] Feature: admin može da odobri submitted team
- [ ] Feature: admin može da odbije sa razlogom
- [ ] Feature: notifikacije se kreiraju u database channel-u
- [ ] Feature: email log mailer pokazuje slanje (log se može parsovati u test-u)
- [ ] Browser smoke: complete UC5 flow end-to-end

## Shared edit zones
- `routes/teams.php` — koordinacija sa T2.1a (forma rute) i T2.1b (certifikati rute)
- `routes/admin.php` — submit / approve admin rute
- `lang/me/teams.php` — koordinacija sa T2.1a i T2.1b
- `app/Notifications/` — **isključivo** ovaj track za team-related notifications

## Acceptance criteria
- Sve Pest testove pass (sva 3 alt toka UC5)
- Browser smoke: profesor radi cijeli UC5 flow bez greške
- Pint clean, Wayfinder regen
- Audit log za team.submitted, team.approved, team.rejected, signature.invalid
- Notifikacije vidljive u DB (`notifications` tabela) i log file
